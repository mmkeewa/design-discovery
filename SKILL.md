# design-discovery

Гид по сбору референсов для дизайн-задачи. Skill идёт по трём слоям последовательно, на каждом — пауза, чтобы ты выбрал, что тебе нравится. В финальный мудборд попадают только твои выборы.

## Как пользоваться

В чате напиши:
> "Опиши свой кейс и какие у него есть компоненты, потом запусти design-discovery."

И дальше опиши задачу в свободной форме. Пример:
> "Делаю онбординг для финтех-приложения. ЦА — gen-z. Компоненты: 4-шаговый wizard с прогресс-баром, биометрика на последнем шаге, motivational копирайтинг."

Skill принимает любой формат — список, абзац, голосовая расшифровка. Главное — указать **тип задачи** и **какие компоненты ты уже знаешь, что будут**.

---

## Что делает skill

Три слоя источников, в строгом порядке:

| Слой | На какой вопрос отвечает | Источники |
|---|---|---|
| 1. Patterns | Как обычно решают этот флоу? | Mobbin, PageFlows, Liba, Benchmarkee |
| 2. Visual | Как сделать запоминающимся? | Awwwards, Godly, Screen Gallery, FPS60 |
| 3. Mood | Расширить настроение | Pinterest, Behance, Dribbble, UI8 |

**После каждого слоя** skill останавливается и спрашивает: *"Что из этого тебе нравится?"* Ты отвечаешь номерами или именами. Skill сохраняет твой выбор и переходит к следующему слою.

**Слой 3 — опциональный.** После Layer 2 skill спросит, нужно ли расширение mood'а, или сразу собирать мудборд из того, что есть.

## Процесс

### Шаг 0 — проверка Claude in Chrome

Перед запуском убедись, что Claude in Chrome MCP подключён. Если нет — skill **не работает**: все источники прячут контент за JS, без браузерной автоматизации это пустые ссылки.

Если MCP не подключён — честно скажи пользователю: "Для design-discovery нужен Claude in Chrome MCP. Подключи его и перезапусти."

---

### Шаг 1 — парсинг брифа и компонентов

Пользователь даёт бриф + список компонентов. Загрузи `reference/brief-parser.md`. Извлеки:

1. **Тип экрана / флоу**
2. **Индустрия**
3. **UI-паттерны** (с приоритетом тех, что пользователь явно перечислил в "компонентах")
4. **Визуальный тон**
5. **Платформа**

Покажи пользователю распознанную структуру **и спроси подтверждение**:

```
Поняла бриф как:
- Тип: [flow]
- Индустрия: [industry]
- Компоненты: [список] ← взяла из твоего описания
- Визуальный тон: [tone] (предполагаю)
- Платформа: [platform]

Правильно? Если нет — поправь, и запускаю Layer 1.
```

**Не запускай Layer 1 без подтверждения.** Это критично — иначе все три слоя промахнутся.

---

### Шаг 2 — Layer 1: Patterns

Загрузи `reference/source-urls.md`, секция Layer 1. Через Claude in Chrome последовательно (или batch'ем, если так быстрее) пройди по четырём источникам:

1. **Mobbin** — `https://mobbin.com/search/screens?keyword={flow}`
2. **PageFlows** — `https://pageflows.com/?search={flow}`
3. **Liba** — `https://liba.app/components/{pattern}` 
4. **Benchmarkee** — `https://benchmarkee.com/search?q={flow}+{industry}`

С каждого источника собери:
- Название компании / приложения
- Название экрана или флоу
- URL
- Превью-скриншот (через `get_page_text` + `find`, или скриншот страницы)
- Одну строку: **что взять** (структура / motion / иерархия / копирайтинг)

**Что показать пользователю:**

```
LAYER 1 — PATTERNS

Нашла 14 ссылок. Вот они:

[1] Revolut — Onboarding flow 2024
🔗 https://mobbin.com/...
📌 Структура 5 шагов с большим прогресс-баром

[2] Cash App — First-launch
🔗 https://mobbin.com/...
📌 Минимум полей, один фокальный CTA

[3] Step — Teen fintech onboarding (PageFlows)
🔗 https://pageflows.com/...
📌 Разговорный копирайтинг, "Hey, what do you want to call your money?"

[...все 14 примеров...]

Что из этого тебе нравится? Ответь номерами через запятую (например "1, 3, 7") 
или скажи "все" / "ничего, ищем другое".
```

**Дождись ответа пользователя.** Сохрани выборы.

Если пользователь говорит "ничего, ищем другое" — спроси, что именно не подходит, скорректируй parsed brief, перезапусти Layer 1. Не больше двух перезапусков — на третий честно скажи: "Бриф слишком абстрактный для этих источников, попробуй уточнить тип задачи."

---

### Шаг 3 — Layer 2: Visual

После того как пользователь сделал выбор на Layer 1, **в одном следующем сообщении** запусти Layer 2.

Загрузи `reference/source-urls.md`, секция Layer 2. Через Claude in Chrome (`browser_batch` параллельно) пройди по:

1. **Awwwards** — фильтр по визуальному тону + категория
2. **Godly** — фильтр по тегам тона
3. **Screen Gallery** — категория по типу экрана
4. **FPS60** — для motion-моментов

С каждого собери:
- Название / автор
- URL
- Скриншот превью
- Одну строку: **что взять** (типографика / композиция / motion / цвет / микро-взаимодействие)

**Что показать пользователю:**

```
LAYER 2 — VISUAL

Нашла 12 визуальных референсов. Вот они:

[1] Linear marketing site (Godly)
🔗 https://...
🖼️ [preview]
📌 Display-typography с тонким контрастом, gradient как фон шага

[2] Arc browser onboarding (Awwwards)
🔗 https://...
🖼️ [preview]
📌 Motion — каждый шаг appears через slide+scale

[...все 12 примеров...]

Что из этого тебе нравится? Ответь номерами.
```

Дождись ответа. Сохрани выборы.

---

### Шаг 4 — развилка перед Layer 3

После того как пользователь выбрал из Layer 2, **остановись и спроси**:

```
Layer 1 + Layer 2 готовы. Из твоих выборов вижу направление:
[одна-две строки — куда тяготеют выборы]

Хочешь:
1. Расширить mood (Pinterest / Dribbble / Behance / UI8) — Layer 3
2. Собрать мудборд из того, что есть
3. Перезапустить с другим брифом
```

Если 1 → Layer 3. Если 2 → сразу к Шагу 6 (сборка мудборда). Если 3 → вернуться к Шагу 1 с новым брифом.

**Важно:** не запускай Layer 3 автоматически. Он широкий и часто шумный — пользователь должен явно сказать "хочу mood".

---

### Шаг 5 — Layer 3: Mood (опционально)

Загрузи `reference/source-urls.md`, секция Layer 3. Через Claude in Chrome пройди по:

1. **Pinterest** — нужен залогиненный браузер. Если не залогинен, **скажи об этом** и предложи Behance fallback
2. **Dribbble** — также часто требует login, но многое доступно и без
3. **Behance** — открытая выдача, обязательно с фильтрами по `ui-ux` field
4. **UI8** — посмотри готовые kit'ы, которые передают общий vibe (не для копирования, а для настроения)

С каждого собери:
- Превью / скриншот
- URL
- Автор / название (без "что взять" — это уже mood, не функциональный)

**Что показать пользователю:**

```
LAYER 3 — MOOD

Нашла 10 пинов / shots / проектов. Вот они:

[1] Pinterest pin
🔗 https://...
🖼️ [preview]

[2] Dribbble shot by @author
🔗 https://...
🖼️ [preview]

[...]

Что из этого расширяет настроение? Номерами.
```

Дождись ответа.

---

### Шаг 6 — финальный мудборд

Загрузи `reference/moodboard-template.md`. Собери markdown **только из того, что пользователь выбрал** на каждом слое.

Сохрани в `outputs/{brief-slug}-moodboard.md`.

Покажи пользователю:

```
Мудборд собран → outputs/{brief-slug}-moodboard.md

В нём:
- Distilled brief
- Layer 1: [N] выбранных паттернов
- Layer 2: [N] выбранных визуалов  
- Layer 3: [N] mood-расширений (или пропущено)
- Synthesis: [одна строка про направление]
- Чек-лист "что дальше"

Хочешь:
1. Открыть и пройтись по ссылкам
2. Собрать всё в FigJam-доску (нужен Figma MCP)
3. Запустить новый бриф
```

## Что skill НЕ делает

- Не генерирует дизайн — это `figma-generate-design`
- Не делает критику референсов — это `impeccable critique`
- Не курирует автоматически — курация на твоей стороне
- Не работает без Claude in Chrome MCP

## Подводные камни

- **Mobbin и PageFlows** имеют paywall на детали флоу. Skill собирает с listing-страниц то, что видно бесплатно (название, превью, базовое описание). Полные видео и кейсы требуют подписку — это нормально, skill даёт ссылку, ты сам решаешь
- **Pinterest и Dribbble** прячут контент за login/JS. Если твой Chrome не залогинен — skill вернёт мало или ничего. Honest fail → переходи на Behance
- **Awwwards** часто показывает B2B-сайты как сайт недели. Если бриф про мобильный продукт — фильтруй жёстко
- **UI8** — это маркетплейс готовых kit'ов. Берётся для mood, не для копирования

## Когда вызывать другие skills

- После discovery, если нужна реализация → `figma-generate-design`
- Если бриф размытый и нужно сначала понять, что вообще делать → `process-audit` (в работе)
- Если хочется ревью собранного мудборда → `impeccable critique`
