# layer-flow.md

Sequential flow с user-curation паузами. Это сердце skill'а — главное отличие от auto-моды.

## Основной принцип

Skill **не курирует автоматически**. Skill приносит результаты, **показывает пользователю**, **спрашивает, что нравится**, **сохраняет выбор**, переходит к следующему слою.

В финальный мудборд попадают только пользовательские выборы.

### Возможные ответы пользователя

**Ответ: "1, 4, 7, 8" (выбор)**
- Сохрани выборы в скрытое состояние skill'а
- Переходи к Layer 2 без дополнительных вопросов

**Ответ: "все"**
- Сохрани все 14 ссылок
- Переходи к Layer 2
- Предупреди: "Сохраню все, но в финальный мудборд возьму топ-8 по релевантности. Если хочешь отжать дальше — скажи."

**Ответ: "ничего"**
- Спроси: "Что не подходит? Слишком корпоративно? Не та индустрия? Не тот платформ?"
- На основе ответа скорректируй parsed brief
- Перезапусти Layer 1 (не больше 2 перезапусков подряд)

**Ответ: "ещё ищем"**
- Аналогично "ничего" — спрашивай, чего не хватает

**Ответ "что-то странное" (например, цитата вместо номера):**
- Уточни: "Не совсем понятно. Дай номера через запятую."

### Развилка перед Layer 3

После выбора на Layer 2, **остановись и спроси**:

```
Layer 1 + Layer 2 готовы. 
Из твоих выборов вижу направление:
"Display typography + один тёплый accent + минимум полей + разговорный copywriting"

Хочешь:
1. Расширить mood через Pinterest / Dribbble / Behance / UI8
2. Собрать мудборд из того, что есть
3. Перезапустить с другим брифом
```

**Hard rule:** Layer 3 не запускается автоматически.

### Sanity check на пустые/слабые слои

Если в каком-то слое менее 5 ссылок:

```
LAYER 1 — PATTERNS
Прошлась по 4 источникам. Mobbin вернул только 2 примера, PageFlows — 1.

Найдено: всего 4 ссылки.

[список]

Это мало для нормальной курации. Варианты:
1. Идти дальше с этими 4 (Layer 2 может скомпенсировать)
2. Расширить запрос — снять фильтр индустрии или конкретного компонента
3. Перезапустить с другим брифом

Что делаем?
```

Никогда не молчи о пустых результатах. Honest fail > silent failure.

## State между слоями

Skill держит в памяти контекста:

```
state = {
  brief: { parsed brief object },
  layer_1: {
    found: [list of all 14],
    selected: [1, 4, 7, 8]  ← из ответа пользователя
  },
  layer_2: {
    found: [list of all 12],
    selected: [2, 5, 7]
  },
  layer_3: {
    found: [list of all 10],
    selected: [3, 9]
  }
}
```

Этот state используется в `moodboard-template.md` для сборки финального файла.

## Если пользователь хочет вернуться к предыдущему слою

Допустимо. Пример:

> "Подожди, на Layer 1 я хочу ещё [3]."

Ответ:
> "Добавила [3] в Layer 1 selections. Текущий выбор: 1, 3, 4, 7, 8. Продолжаем с Layer 2 или переснимать?"

Это live-курация, и она важна. Пользователь может передумать, и skill это поддерживает.
