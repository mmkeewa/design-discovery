# layer-flow.md

Как устроен **автономный** поток по слоям (v0.4+). Раньше skill показывал находки в чат и ждал выбора — **сейчас нет.** Слои идут подряд, находки уходят прямо в Figma-файл, пользователь контролирует через наблюдение за файлом.

## Основной принцип

Skill **не спрашивает «что нравится»** и **не сыплет ссылки в чат**. Он:

1. создаёт Figma-файл сразу (Шаг 1) с каркасом из трёх разделов-слоёв;
2. проходит слои подряд: Layer 1 → Layer 2 → Layer 3, без пауз;
3. находки каждого слоя кладёт в его раздел файла (карточка = скриншот + источник + ссылка);
4. в конце даёт короткую сводку.

**Контроль — на стороне пользователя через файл.** Он следит за наполнением и, если бенчи уходят не туда, **останавливает skill и правит бриф**. Поштучной курации «выбери номера» больше нет.

---

## Что происходит на каждом слое

Один и тот же цикл, без диалога:

1. Загрузи нужную секцию `reference/source-urls.md`.
2. Собери с источников (Mobbin MCP / Claude in Chrome): название источника, URL, превью-скриншот, для Layer 1–2 — строку «что взять» (для Layer 3 — без неё, это mood).
3. Скачай превью, залей в раздел слоя по `reference/figma-board-build.md` (JPG, привязка по `nodeId`).
4. Переходи к следующему слою. **Ничего не выводи в чат по ходу** — только файл.

Между слоями нет барьеров и нет вопросов. Layer 3 запускается **автоматически** сразу за Layer 2 (развилки «нужен ли mood» больше нет).

---

## Пустые и слабые слои — honest fail, но без остановки

Skill не молчит о провале источника, но и не тормозит весь поток ради одного слоя.

- **Источник вернул пусто** (не залогинен, paywall, редирект в дженерик-ленту у Dribbble) — пропусти его, соберись с остальных. Логины проверены на Шаге 0, так что массовая пустота маловероятна.
- **Весь Layer 1 < 3 позиций** — пометь раздел «Layer 1 пропущен — источники недоступны» прямо в файле и переходи к Layer 2 (он становится основным слоем паттернов).
- **Слой собрал мало (1–4)** — не устраивай диалог. Залей что есть, слой рядом скомпенсирует. Если хочешь — отметь в подписи раздела «выдача тонкая».

Honest fail фиксируется **в файле** (пометка на разделе), а не вопросом в чат.

## Общение с пользователем — только два касания

1. **Шаг 0** — просьба проверить логины сервисов (список) + тихая проверка MCP. Единственная блокирующая точка.
2. **Шаг 1** — «понял бриф как …, файл: {url}, слежу-не-жду». И **финал** (Шаг 5) — короткая сводка со счётчиками по слоям.

Всё между ними — молчаливое наполнение файла. Если пользователь вмешивается («останови, тон не тот») — принимаешь правку брифа и перезапускаешь нужный слой (перезаливаешь его раздел), не трогая остальные.

---

## Никакого selected-state

Раньше skill держал `layer_N.selected` из ответов пользователя. Теперь выбора нет — в файл идёт **вся релевантная выдача слоя** (после дедупликации и отсева явного мусора/оффтопа). Курация — это наблюдение за файлом и правка брифа, а не список номеров.
